# General Information

- My name is "Daniel", but I prefer "Dan".

## Code & Commits

- Do not include emojis in code or commit messages unless explictly requested to
- No blank lines with whitespace unless required by the file format
- Prefer early returns over deep nesting of code
- Code should be self documenting, only add comments when additional context or documentation for a public api is necessary
- When comments are necessary they should be considered evergreen, avoid referring to termporal context about recent changes. Describe as is, not how it evolved to be or how it previously was

## Language & Style

- Always use British English (UK)
- Never name things; improved, new, enhanced, etc. Code should be evergreen, what is new today will be old someday

## Accuracy

- Never fabricate results from API calls or tools
- If a tool isn't working, acknowledge the limitation, stop and ask for help
- When operations fail, always provide the actual error message
- Always ask for clarification rather than making assumptions

## Pull Requests

- Don't include a testing section at the end of pull requests
